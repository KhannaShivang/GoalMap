# AI Goal Planner

An AI-powered learning roadmap generator that creates personalized step-by-step plans to help users achieve their career goals. Built with Java, Spring Boot, Spring AI, PostgreSQL, and pgvector.

---

## What it does

A user enters their current skills and a career goal. The system calls an LLM (Groq/llama-3.1) to generate a structured 10-step roadmap. Each task has subtasks breaking it into concrete learning steps, and a quiz at the end to test understanding. As the user progresses, the AI adapts the roadmap based on completed tasks.

**Example:**

> Goal: *"Become a backend developer at a product company in 12 months"*
>
> Current skills: *"Final year CS student, knows Java and DSA, solved 200 LeetCode problems"*

The system generates:

1. Master Java collections and OOP principles
2. Learn Spring Boot and build your first REST API
3. Understand databases — SQL, PostgreSQL, JPA
4. Learn Docker and containerisation
5. Study system design fundamentals
6. ... and 5 more tailored steps

Each task expands to show subtasks (e.g. "Learn if/else", "Learn loops", "Learn switch statements") and ends with a 5-question quiz.

## Features

### AI roadmap generation

Sends a structured prompt to Groq with the user's profile and goal. The LLM returns 10 ordered learning steps mapped to specific skills. Tasks are saved to the database and returned immediately.

### Subtasks (on demand)

When a user opens a task, the AI generates 4–6 concrete subtasks breaking the topic into specific things to learn. Generated once, cached in the database forever after.

### Adaptive feedback loop

After completing 3+ tasks, users can trigger recalculation. The AI looks at what's been completed and generates 5 new steps tailored to their progress — skipping anything already done.

### Quiz system

When all subtasks for a task are complete, a 5-question multiple choice quiz is generated by the AI. Answers are submitted and scored server-side. Explanations are returned for every question. Pass mark is 60%.

### Vector search (RAG)

Learning resources are embedded using `nomic-embed-text` and stored as vectors in pgvector. Similarity search finds the most relevant resources for any skill query using cosine distance.

### JWT authentication

All endpoints are protected. Signup returns a JWT token valid for 24 hours. The token is sent as a Bearer header on every subsequent request.

## API reference

### Auth

| Method | Endpoint             | Description                 |
| ------ | -------------------- | --------------------------- |
| POST   | `/api/auth/signup` | Create account, returns JWT |
| POST   | `/api/auth/login`  | Login, returns JWT          |

### Users

| Method | Endpoint            | Description                      |
| ------ | ------------------- | -------------------------------- |
| POST   | `/api/users`      | Create user (legacy, use signup) |
| GET    | `/api/users/{id}` | Get user by ID                   |
| PUT    | `/api/users/{id}` | Update user profile              |

### Goals

| Method | Endpoint                   | Description              |
| ------ | -------------------------- | ------------------------ |
| POST   | `/api/goals`             | Create a new goal        |
| GET    | `/api/goals/{id}`        | Get goal by ID           |
| GET    | `/api/goals?userId={id}` | Get all goals for a user |

### Roadmaps

| Method | Endpoint                      | Description               |
| ------ | ----------------------------- | ------------------------- |
| POST   | `/api/roadmaps/generate`    | Generate AI roadmap       |
| GET    | `/api/roadmaps/{id}`        | Get roadmap with tasks    |
| GET    | `/api/roadmaps?userId={id}` | Get all roadmaps for user |

### Tasks + Progress

| Method | Endpoint                           | Description                   |
| ------ | ---------------------------------- | ----------------------------- |
| PUT    | `/api/tasks/{id}/complete`       | Mark task complete/incomplete |
| GET    | `/api/roadmaps/{id}/progress`    | Get progress summary          |
| POST   | `/api/roadmaps/{id}/recalculate` | AI generates next steps       |

### Subtasks

| Method | Endpoint                            | Description                |
| ------ | ----------------------------------- | -------------------------- |
| GET    | `/api/tasks/{id}/subtasks`        | Get or generate subtasks   |
| PUT    | `/api/subtasks/{id}/complete`     | Mark subtask complete      |
| GET    | `/api/tasks/{id}/subtasks/status` | Check if all subtasks done |

### Quiz

| Method | Endpoint                        | Description               |
| ------ | ------------------------------- | ------------------------- |
| GET    | `/api/tasks/{id}/quiz`        | Get or generate quiz      |
| POST   | `/api/tasks/{id}/quiz/submit` | Submit answers, get score |

### Resources

| Method | Endpoint                         | Description                 |
| ------ | -------------------------------- | --------------------------- |
| GET    | `/api/resources/{skillId}`     | Get resources for a skill   |
| GET    | `/api/resources/search?query=` | Vector similarity search    |
| POST   | `/api/resources/embed`         | Embed all pending resources |

## Local setup

### Prerequisites

- Java 17+
- Maven 3.8+
- PostgreSQL 15+
- pgvector extension
- Ollama (for embeddings)
- Groq API key (free at console.groq.com)

### 1 — Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/ai-goal-planner.git
cd ai-goal-planner
```

### 2 — Set up PostgreSQL

```bash
# Create database
psql -U postgres -c "CREATE DATABASE goalplanner;"

# Enable pgvector
psql -U postgres -d goalplanner -c "CREATE EXTENSION IF NOT EXISTS vector;"

# Run schema
psql -U postgres -d goalplanner -f backend/src/main/resources/schema.sql

# Run migration (subtasks + quiz tables)
psql -U postgres -d goalplanner -f backend/src/main/resources/migration_subtasks_quiz.sql
```

### 3 — Install Ollama and pull embedding model

```bash
# Install from https://ollama.com/download
ollama pull nomic-embed-text

# Start Ollama (keep running in background)
ollama serve
```

### 4 — Configure environment

Set your Groq API key as an environment variable:

```bash
# Linux / macOS
export GROQ_API_KEY=gsk_your_key_here

# Windows PowerShell
$env:GROQ_API_KEY = "gsk_your_key_here"
```

### 5 — Build and run

```bash
mvn clean install
mvn spring-boot:run
```

The API is now running at `http://localhost:8080`

### 6 — Open the frontend

Open `frontend/index.html` in your browser, or use VS Code Live Server.
